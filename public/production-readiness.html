Production readiness

What should be configured before real paid customer use.

This page is for owner/staff review. The platform can run as an MVP, but sensitive legal, tax, medical, family, financial, and government-document support needs careful configuration before public paid traffic.

Required before charging real users

  • Set a long private ADMIN_TOKEN.
  • Set APP_BASE_URL to the live domain.
  • Configure SMTP so owner and user confirmations can be sent.
  • Configure Stripe secret key and review price IDs.
  • Use Postgres plus persistent/object upload storage before sensitive production use.
  • Set tawk.to or another live chat tool if live support is advertised.
  • Confirm public pricing, refund/cancellation language, and what each paid review includes.

Built into the current foundation

  • Security headers and public-route rate limiting foundation.
  • Stripe Checkout foundation with success/cancel pages and webhook handling.
  • Optional Postgres-backed JSON persistence foundation.
  • Render persistent-disk path support for upload storage.
  • SMTP email delivery scaffolding with local notification spool fallback.
  • Longer secure continuation links for saved files.
  • Human Review Specialist workbench and audit log foundation.
  • Downloadable organized file summary and form-preparation starter file.
  • Official-source tracking and missing-information checks for supported form paths.
  • Clear separation between internal human review and attorney/tax/professional review.

Trust boundary

Smarter Justice is not a law firm and not the government. Official government forms are often free from the government. Users remain responsible for reviewing, signing, filing, and submitting any official forms unless a separate professional engagement says otherwise.
