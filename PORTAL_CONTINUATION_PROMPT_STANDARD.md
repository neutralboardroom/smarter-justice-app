# Smarter Justice Portal Continuation Prompt Standard

Version 1.3.3

Every focused portal handoff prompt must preserve both the current Smarter Justice shared-system rules and the complete portal-specific source of truth. Generic ecosystem language must never erase specialty requirements, and specialty history must not weaken shared privacy, security, truthfulness, source verification, consent, or professional boundaries.

## Automatic Master Rules Pack inheritance

Every continuation prompt, replacement prompt for a long or unstable chat, manifest, and build handoff automatically embeds or references the current Master Rules and Suggestions Pack with version and SHA-256 checksum. The owner should not need to request this again.


## Professional directory and acquisition evidence

When a portal uses the central professional marketplace, its handoff must record the default directory page size, pagination or show-more behavior, individual and firm profile coverage, public-source status, claim and account-control tests, campaign attribution, membership checkout status, and narrow-phone acquisition review. A successful single-record test is not sufficient evidence that a multi-record directory or firm finder works.

## Official-source and onboarding evidence

Where a portal uses public records, professional profiles, or a marketplace, the handoff must record source availability behavior, default result size, offset or cursor pagination, filter preservation, clear controls, public URL safety, profile metadata, claim continuity, onboarding checklist status, mobile autofill review, firm pricing transparency, and free-versus-paid pricing clarity.

## Required sections

1. Prompt priority and newest-ZIP/code source-of-truth rules.
2. Exact package roles: full/archive, lean continuation, overlay, complete deployable, historical, and do-not-clean-deploy requirements.
3. Master Rules Pack version, checksum, protected API reference, embedded rules, and change history.
4. Portal identity, mission, audience, scope, exclusions, public boundaries, and specialty vocabulary.
5. Smarter Justice operating model, Master Coordination responsibilities, and separate portal-chat rule.
6. Shared standards and explicit portal-specific deviation authority.
7. Portal-specific workflows, forms, jurisdictions, pricing, AI, referrals, dashboards, roles, security, professional marketplace, revenue, and deployment requirements.
8. Current production/development versions, latest package, repository, deployment, domain, health, environment, and current live-verification status.
9. Completed capabilities that must be preserved.
10. Partial foundations, mocked integrations, known limitations, future features, and unresolved issues.
11. Owner decisions and reasons.
12. Documented shared-standard adaptations with default, reason, scope, risk, mitigation, and benefit.
13. Professional types, central account participation, public-profile source plan, claims, credentials, membership, firm seats, consultation/services, consent, sponsorship, reviews, and revenue requirements.
14. Current test evidence, form/source ledgers, manual QA, launch cohort, capacity, first-case follow-up, and go/no-go status.
15. Risks, blockers, required credentials, legal/professional review, and launch readiness.
16. Recommended next build and ordered workflow.
17. Required tests, fresh-unzip verification, clean storage, release notes, manifest, Control Center summary, prompt refresh, and final ZIP.
18. Instruction not to break working functionality or change merely for the sake of changing.

## Prompt completeness rule

A Control Center-generated prompt is a governed starting point. Before use, the active portal chat must merge the newest complete specialty decisions, forms, workflows, version history, code findings, unresolved issues, and deployment facts. The latest complete prompt explicitly approved by the owner remains authoritative unless deliberately superseded.

## Professional acquisition inheritance

Every relevant portal prompt preserves public-source seeding, truthful unclaimed status, central Smarter Justice profile management, owner-approved claims, paid membership eligibility, NYC founding-member pilot, Stripe-confirmed subscriptions, mobile/QR enrollment, firm per-seat billing and discounts, campaign attribution, and strict separation of payment from AI analysis, verification, ratings, and neutral eligibility. State any justified specialty adaptation.
